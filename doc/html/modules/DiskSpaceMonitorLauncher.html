DiskSpaceMonitorLauncher Module¶

class DiskSpaceMonitorLauncher.DiskSpaceMonitorLauncher(gconfig, dconfigs, logger=None)¶

Bases: object

A launcher which starts and stops different DiskSpaceMonitor daemons for each filesystem on which data_stream directories live. Note that it doesn’t test which filesystem incoming directories live on; these are assumed to be on the same filesystem as the data_stream directory for that data_stream.

getDescription(filesys)¶

Get short and long descriptions for a given filesystem (input string is name of the mount point) long to be used for log messages + emails short to be used for tags on log messages and for process naming

groupDatasetsByDisk()¶

Return a dictionary containing the data_streams for each file system: key is the mount point, value is an array of data_stream config objects. Use this so that we can launch a separate DiskSpaceMonitor for each filesystem.

kill()¶

Kill all DiskSpaceMonitor daemons that have been launched

killOne(daemon)¶

Kill one daemon. Argument is a DaemonCtl object.

launch()¶

Launch all the DiskSpaceMonitor daemons that are required

launchOne(ds_filesys, ds_group)¶

launch a single DiskSpaceMonitor for a group of data_stream that reside on a given filesystem.

inputs: ds_filesys (string) - name of mount point
ds_group - list of DatasetConfig objects just for
this group of data_streams
makeUnique(tag)¶

Given a tag, append prefixes as required so that it is unique. Reason for this is so that if more than one filesystem has the same short name (last pathname element of the mount point), they will have distinct tags for logging and process naming purposes.
